# v1.5.3 Booking Consent Checkboxes — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add required email + SMS consent checkboxes to the booking form, the post-booking quick-register form, and the standalone account-page register form. Make phone required everywhere. Persist consent on the booking row (new DB columns) and on the WP user account (user meta). Wording is settings-configurable with `{business_name}` interpolation. Reschedule flow inherits consent from the original booking.

**Architecture:** Two new settings (`email_consent_text`, `sms_consent_text`) read via a new pure helper `caswell_render_consent_text()`. Three new bookings-table columns gated by a `BOOKINGS_VERSION` bump. Three forms updated symmetrically — same checkbox markup, same payload keys (`email_consent`, `sms_consent`), same server-side enforcement pattern. Reschedule flow detects the original booking and inherits the original's consent flags + timestamp.

**Tech Stack:** WordPress 6.x, PHP 8.1+, jQuery, PHPUnit 9.6 + Brain Monkey 2.6 for unit tests. No new runtime dependencies.

Create `includes/consent.php`:

```php
<?php
/**
 * Consent-text rendering.
 *
 * Returns the email or SMS consent line with {business_name}
 * interpolated. Defaults are TCPA-style transactional consent
 * for confirmation + reminder messages.
 */

defined( 'ABSPATH' ) || exit;

function caswell_consent_text_defaults() {
    return [
        'email' => 'I agree to receive appointment confirmation and reminder emails from {business_name} at the email address provided.',
        'sms'   => 'I agree to receive appointment confirmation and reminder text messages from {business_name} at the phone number provided. Message frequency varies. Msg & data rates may apply. Reply STOP to opt out.',
    ];
}

function caswell_render_consent_text( $kind ) {
    $defaults = caswell_consent_text_defaults();
    if ( ! isset( $defaults[ $kind ] ) ) {
        return '';
    }
    $template = caswell_get_option( "{$kind}_consent_text", '' );
    if ( '' === $template ) {
        $template = $defaults[ $kind ];
    }
    $business = caswell_get_option( 'business_name', '' );
    if ( '' === $business ) {
        $business = get_bloginfo( 'name' );
    }
    return str_replace( '{business_name}', $business, $template );
}
```

## Notes for the implementer

- **Tests over WP**: there's no local WordPress install. PHPUnit + Brain Monkey runs pure-PHP — no `wp-cli`, no DB. If a step says "run PHPUnit" and you don't have `vendor/`, run `composer install` once.
- **No build step**: there's nothing to compile. The plugin loads directly from the source tree.
- **Frequent commits**: each task is a single commit. Don't squash. If you make a mistake mid-task, fix it inside the same commit (use staged adds, not amending across tasks).
- **Don't reach for unused helpers**: if you find a place that "could also" use the new helper or the new columns, leave it alone — out of scope for v1.5.3.
- **Reschedule path coverage**: when in doubt about the reschedule branch, the rule is: client never sees the checkboxes; server inherits the original booking's `email_consent`, `sms_consent`, and `consent_timestamp` verbatim.
- **One-shot smoke**: after Task 11, before Task 12, manually open `caswell-booking.php` and `includes/class-booking-handler.php` once and grep for `email_consent` — there should be hits in both. Cheap sanity check before tag.
