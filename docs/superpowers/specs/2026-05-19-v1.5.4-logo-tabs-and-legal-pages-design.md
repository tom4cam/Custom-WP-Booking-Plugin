# v1.5.4 — Site logo + Bookings tab first + Privacy/Terms pages — Design

**Date:** 2026-05-19
**Status:** Approved
**Scope:** Three changes shipping together:

1. Add an admin-configurable site logo (WP media library picker) that
   renders on the home nav, booking widget, confirmation/reminder
   emails, and the admin settings page.
2. Reorder the admin tabs so **Bookings** is the leftmost tab and the
   default view.
3. Auto-create **Privacy Policy** and **Terms of Use** pages on first
   activation with sensible plain-HTML defaults (edits cleanly in
   either the Classic Editor or the Block Editor), and surface
   links to both on the home footer, booking widget, email footers,
   and account page.

## Goal

1. Branding: one logo, four contexts, white-label friendly.
2. Operational ergonomics: land directly on the daily-use Bookings list
   when opening Settings → Caswell Booking.
3. Compliance: every client-facing surface has visible Privacy Policy
   and Terms of Use links; the pages themselves arrive with reasonable
   defaults Tom can edit (and should have a lawyer review eventually).

## Non-goals

- Multiple logo variants (light/dark, mobile/desktop). One image; CSS
  scales it per context.
- A favicon setting. WP has site-icon support already.
- A cookie banner / GDPR consent gate. Out of scope; not legally
  required for a US single-state massage business.
- Auto-updating the legal page text when the plugin updates. Once Tom
  edits the pages, his edits stay — the plugin only creates the pages
  on first run.
- WP's built-in "Privacy Policy page" registration. The plugin creates
  its own page so the surfaced links always work even on a site whose
  WP-level Privacy Policy slot is unset.

## Part 1 — Logo support

### New settings

Two new keys on the existing `caswell_settings` blob (no schema change):

| Key | Type | Sanitizer |
|---|---|---|
| `branding_logo_url` | URL string | `esc_url_raw` |
| `branding_logo_id` | int (attachment ID) | `absint` |

Both default to empty. `branding_logo_url` is the source of truth at
render; `branding_logo_id` exists only so the media picker can restore
its "selected" state on page reload.

### Admin UI — Business Info tab

New row in the existing Business Info table:

```php
<tr>
    <th><label>Site Logo</label></th>
    <td>
        <div class="caswell-logo-picker">
            <input type="hidden" id="branding_logo_url" name="caswell_settings[branding_logo_url]" value="<?php echo esc_attr( $o['branding_logo_url'] ?? '' ); ?>" />
            <input type="hidden" id="branding_logo_id"  name="caswell_settings[branding_logo_id]"  value="<?php echo esc_attr( $o['branding_logo_id']  ?? '' ); ?>" />

                    <img src="<?php echo esc_url( $o['branding_logo_url'] ); ?>" alt="" style="max-height:80px;max-width:300px;" />

                <?php endif; ?>
            </div>
            <p>
                <button type="button" class="button" id="caswell-logo-choose">Choose Logo</button>

            </p>
            <p class="description">Shown on the home page, booking widget, confirmation/reminder emails, and this admin page. Recommended: a transparent-background PNG or SVG sized around 200×60.</p>
        </div>
    </td>

```

A small JS file (`admin/admin-logo-picker.js`) wires the buttons to
the WP media picker. `class-admin.php::enqueue_admin_assets()` adds
`wp_enqueue_media()` and registers the script — guarded by the
existing plugin-screen check.

```js
jQuery(function ($) {
    var frame;
    $('#caswell-logo-choose').on('click', function (e) {
        e.preventDefault();
        if (frame) { frame.open(); return; }
        frame = wp.media({
            title:    'Choose Logo',
            button:   { text: 'Use as Logo' },
            library:  { type: 'image' },
            multiple: false,
        });
        frame.on('select', function () {
            var att = frame.state().get('selection').first().toJSON();
            $('#branding_logo_url').val(att.url);
            $('#branding_logo_id').val(att.id);
            $('.caswell-logo-preview').html(
                '<img src="' + att.url + '" alt="" style="max-height:80px;max-width:300px;" />'
            );
            $('#caswell-logo-remove').show();
        });
        frame.open();
    });
    $('#caswell-logo-remove').on('click', function (e) {
        e.preventDefault();
        $('#branding_logo_url').val('');
        $('#branding_logo_id').val('');
        $('.caswell-logo-preview').html('<span class="description">No logo selected.</span>');
        $(this).hide();
    });
});
```

## Part 3 — Privacy Policy + Terms of Use pages

### Two new auto-created pages

`caswell_create_pages()` is extended with two new blocks, mirroring
the existing Home / Booking / Account page pattern. Both follow the
"only create if option is unset" guard so Tom can delete and
re-trigger by clearing the option.

```php
if ( ! get_option( 'caswell_privacy_page_id' ) ) {
    $page_id = wp_insert_post( [
        'post_title'   => 'Privacy Policy',
        'post_name'    => 'privacy-policy',
        'post_status'  => 'publish',
        'post_type'    => 'page',
        'post_content' => caswell_legal_default_content( 'privacy' ),
    ] );
    if ( $page_id && ! is_wp_error( $page_id ) ) {
        update_option( 'caswell_privacy_page_id', $page_id );
    }
}

if ( ! get_option( 'caswell_terms_page_id' ) ) {
    $page_id = wp_insert_post( [
        'post_title'   => 'Terms of Use',
        'post_name'    => 'terms-of-use',
        'post_status'  => 'publish',
        'post_type'    => 'page',
        'post_content' => caswell_legal_default_content( 'terms' ),
    ] );
    if ( $page_id && ! is_wp_error( $page_id ) ) {
        update_option( 'caswell_terms_page_id', $page_id );
    }
}
```

If the `privacy-policy` slug is already taken on the site (e.g. WP
auto-created one when the site was first set up), `wp_insert_post()`
suffixes it (`privacy-policy-2`). That's WP default behavior; the
option still gets stored against whatever ID was created.

### Default content — plain HTML

Both pages get content stored as **plain HTML** (`<h2>`, `<p>`, `<ul>`,
`<a>`). This edits cleanly in the Classic Editor (the editor Tom is
planning to use) with no block-comment markers cluttering the source
view. In the modern Block Editor the same content renders as a single
"Classic" block, which can be converted to individual blocks via the
block toolbar if anyone wants per-block editing.

A new helper `caswell_legal_default_content( $kind )` in
`includes/legal-pages.php`:

- Reads templates from `templates/legal-privacy-policy.html` and
  `templates/legal-terms-of-use.html` (plain HTML — no block
  delimiters).
- Interpolates these placeholders before returning:
  - `{business_name}` → `caswell_get_option( 'business_name', get_bloginfo( 'name' ) )`
  - `{practitioner_name}` → `caswell_get_option( 'practitioner_name', '' )`
  - `{service_type}` → `caswell_get_option( 'service_type', 'massage therapy' )`
  - `{contact_email}` → `caswell_get_option( 'business_email', get_bloginfo( 'admin_email' ) )`
  - `{site_url}` → `home_url()`
  - `{state}` → hardcoded "Florida" (Ryan's location) for v1.5.4. Future white-label expansion can promote this to a setting.

The actual draft text covers:

**Privacy Policy** (~700 words)
- Introduction (who, what site)
- What we collect (name, email, phone, consent flags + timestamp,
  payment metadata via Square/Venmo, booking history, optional notes)
- How we use it (confirmations, reminders, scheduling, payment)
- Third-party processors: Square, Twilio, Google Calendar, Venmo,
  WordPress hosting — with one-line description and link out to each
  service's own policy
- SMS specifics: explicit consent at signup, frequency varies, Reply
  STOP to opt out, Reply HELP for help, message & data rates may apply
- Data retention: bookings kept for 7 years for tax purposes, then
  archived/deleted
- Your rights: access, correction, deletion — email `{contact_email}`
- Cookies: minimal — WP login session, anti-CSRF nonces. No third-party
  analytics or ad tracking
- Children: service is not directed at children under 18; minors
  require a parent/guardian present
- Changes: posted on this page; updated `{last_updated_date}` (set at
  page-creation time)
- Contact: `{contact_email}`

**Terms of Use** (~600 words)
- Acceptance (using the site = agreeing)
- Service description ({service_type} appointments)
- Booking and confirmation (selecting a time, receiving a confirmation
  email/SMS, the practitioner reserves the right to decline)
- Payment: Square (card, charged at booking) or Venmo (manual, due
  at session)
- Cancellation policy: free up to 24 hours before; cancellations less
  than 24 hours in advance are non-refundable
- No-show: full session fee charged
- Client conduct: arrive on time, sober, communicate any contraindications,
  sexual contact or harassment ends the session immediately with no refund
- Medical disclaimer: not medical advice; consult a physician for any
  health concerns; disclose injuries, medications, pregnancy
- Right to refuse service: at the practitioner's discretion
- Liability: massage involves contact and pressure; practitioner is
  not liable for injuries arising from undisclosed conditions
- Intellectual property: site content owned by `{business_name}`
- Governing law: State of {state}
- Changes: posted on this page; updated `{last_updated_date}`

Both pages end with a footer line:

> *This document was generated by the booking plugin and has not been
> reviewed by a lawyer. Please consult an attorney before relying on it.*

### URL helper

`includes/legal-pages.php` exports `caswell_legal_page_url( $kind )`:

```php
function caswell_legal_page_url( $kind ) {
    $option = $kind === 'terms' ? 'caswell_terms_page_id' : 'caswell_privacy_page_id';
    $page_id = (int) get_option( $option, 0 );
    if ( $page_id ) {
        $url = get_permalink( $page_id );
        if ( $url ) return $url;
    }
    return home_url( '/' . ( $kind === 'terms' ? 'terms-of-use' : 'privacy-policy' ) );
}
```

### Activation behavior on upgrade

`caswell_create_pages()` runs on activation. For upgrade-in-place
(plugin file replaced, no deactivate/reactivate), `caswell_init()` —
the existing `plugins_loaded` callback — gets a sibling check that
re-runs page creation when either legal page ID is missing:

```php
if ( ! get_option( 'caswell_privacy_page_id' ) || ! get_option( 'caswell_terms_page_id' ) ) {
    caswell_create_pages();
}
```

The existing `caswell_create_pages()` is idempotent — guards prevent
duplicate Home/Booking/Account pages.

## Migration / compatibility

- **No DB schema change.** No `BOOKINGS_VERSION` bump.
- **New options:** `branding_logo_url`, `branding_logo_id`,
  `caswell_privacy_page_id`, `caswell_terms_page_id`. All default empty.
- **Pages on upgrade:** `caswell_init()` triggers `caswell_create_pages()`
  when either legal page ID is missing — covers in-place upgrades.
- **No logo set:** every render context falls back to today's behavior.
- **Existing legal pages:** if a site already has its own
  `/privacy-policy` slug, `wp_insert_post()` will append `-2`. The
  option ID stores the new ID; the surfaced link points to whatever
  was created. Tom can manually fix the slug in WP if he wants.
- **Existing edits stay:** the create-page block only runs when the
  ID option is unset. Once Tom edits, his edits stay.
