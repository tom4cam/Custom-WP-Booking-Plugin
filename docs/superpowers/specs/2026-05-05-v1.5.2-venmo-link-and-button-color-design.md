# v1.5.2 — Venmo payment link in confirmations + email button-color fix — Design

**Date:** 2026-05-05
**Status:** Approved
**Scope:** Five small changes shipping together as v1.5.2.

## Goal

1. When a client books with Venmo and hasn't paid yet, surface a clickable
   "Pay $XX with Venmo" button in the confirmation email and append a
   `Pay via Venmo: <url>` line to the confirmation SMS. The link
   pre-loads the amount and a memo identifying the booking.
2. Fix the "Visit Our Website" button in the confirmation email so its
   styling survives Gmail and other email clients that strip
   `<style>` blocks.

## Non-goals

- Reminder emails (only confirmations).
- Marking the booking paid via the link — Venmo's personal-account
  webhooks aren't usable here; payment status remains manual.
- URL shortening for SMS.
- Configurable Venmo button color, copy, or position.
- Whatsapp formatting differences.

## Helper — `caswell_venmo_payment_link( $booking )`

New function in `caswell-booking.php` next to the other
`caswell_*` helpers. Pure — no side effects. Returns a universal
Venmo web URL (auto-redirects to the app on mobile) or `''`.

```php
function caswell_venmo_payment_link( $booking ) {
    if ( empty( $booking->payment_method ) || $booking->payment_method !== 'venmo' ) return '';
    if ( ! empty( $booking->payment_status ) && $booking->payment_status === 'paid' ) return '';

    $venmo_user = caswell_get_option( 'venmo_username', '' );
    if ( '' === $venmo_user ) return '';
    $venmo_user = ltrim( $venmo_user, '@' );

    $price_key = "venmo_price_{$booking->session_length}";
    $amount    = caswell_get_option( $price_key, '' );
    if ( '' === $amount ) return '';

    $service     = caswell_get_option( 'service_type', 'appointment' );
    $start_ts    = caswell_local_ts( $booking->start_datetime );
    $client_first = strtok( (string) $booking->name, ' ' );
    $last_init   = '';
    $parts       = preg_split( '/\s+/', trim( (string) $booking->name ) );
    if ( count( $parts ) > 1 ) {
        $last_init = ' ' . strtoupper( substr( end( $parts ), 0, 1 ) ) . '.';
    }
    $note = sprintf(
        '%s %dmin — %s — %s%s',
        ucfirst( $service ),
        (int) $booking->session_length,
        wp_date( 'M j, g:ia', $start_ts ),
        $client_first,
        $last_init
    );

    return sprintf(
        'https://venmo.com/%s?txn=pay&amount=%s&note=%s',
        rawurlencode( $venmo_user ),
        rawurlencode( $amount ),
        rawurlencode( $note )
    );
}
```

Note format example: `Massage 60min — May 12, 10:00am — Jane S.`
This goes into the Venmo memo so Ryan can match payments to bookings.

## SMS confirmation — auto-append link

`includes/class-notifications.php`, in `send_sms( $booking, $type )`
(around line 275). After template interpolation, append the Venmo
line when applicable:

```php
private function send_sms( $booking, $type ) {
    if ( empty( $booking->phone ) ) return;

    $template_key = "sms_{$type}_template";
    $template     = caswell_get_option( $template_key, $this->default_sms_template( $type ) );
    $message      = $this->interpolate( $template, $booking );

    if ( 'confirmation' === $type ) {
        $venmo_link = caswell_venmo_payment_link( $booking );
        if ( $venmo_link ) {
            $message .= "\n\nPay via Venmo: {$venmo_link}";
        }
    }

    $this->send_raw_sms( $this->format_phone( $booking->phone ), $message );
}
```

Note: The Venmo URL with note param adds ~80 chars to the SMS, which
may push the message into a second segment (extra Twilio cost). This
is the deliberate tradeoff for one-tap payment.

## AJAX response refactor

`includes/class-booking-handler.php:380-381` currently builds the
booking-widget Venmo link as `venmo://paycharge?...` (mobile-only
app scheme). Replace with the helper output so desktop users (and
the email/SMS recipients) all get the same universal URL.

Before:
```php
'venmo_link' => $venmo_user
    ? "venmo://paycharge?txn=pay&recipients={$venmo_user}&amount={$venmo_price}"
    : '',
```

After:
```php
'venmo_link' => caswell_venmo_payment_link( $booking ),
```

(Reuses the just-loaded `$booking` object that's already fetched
above in the handler.)

## Visit Our Website button — inline styles

`templates/email-confirmation.php` line 68. The existing
`class="cta-btn"` rule lives in a `<style>` block (line 26) which
Gmail and many other email clients strip. Inline the styles:

Before:
```html
<a href="<?php echo esc_url( $site_url ); ?>" class="cta-btn">Visit Our Website</a>
```

After:
```html
<a href="<?php echo esc_url( $site_url ); ?>" style="display:block;width:fit-content;margin:24px auto;background:#4a7c6f;color:#fff;text-decoration:none;padding:14px 32px;border-radius:8px;font-weight:600;text-align:center;">Visit Our Website</a>
```

Then remove the now-unused `.cta-btn` rule from the `<style>` block
on line 26.

The Reschedule and Cancel buttons already use inline styles, so
they're unaffected.

## Migration / compatibility

- No DB schema changes.
- No new settings.
- No removed settings.
- Existing installs: Venmo bookings auto-render the Pay button in
  emails/SMS without any admin action. The booking-widget success
  page now uses the universal URL — improves desktop UX, no
  regression on mobile (mobile auto-opens the app from the web URL).

## Verification

Manual smoke test on the live site after Dropbox sync:

- **Visit Our Website button:** open a fresh confirmation email in
  Gmail (web). The "Visit Our Website" button renders as a green
  pill with white text — not as a plain blue underlined link.
- **Pay with Venmo button (email):** book a session selecting
  Venmo as payment. Confirmation email shows three buttons in a
  row: Pay $XX with Venmo (Venmo blue) | Reschedule (green) |
  Cancel (red border, white). Click the Pay button — Venmo opens
  with the right amount and note pre-loaded.
- **No Pay button when paid:** if the booking is marked paid,
  the Pay button doesn't appear (Reschedule/Cancel still do).
- **No Pay button for Square bookings:** book with Square — no Pay
  button in the email.
- **SMS Pay link:** for a Venmo booking, the SMS includes a
  trailing line `Pay via Venmo: https://venmo.com/...`. Tap on a
  phone — Venmo opens with amount + note pre-filled.
- **Booking widget success page:** book with Venmo on a desktop
  browser. The "Pay with Venmo" button on the success page now
  works (clicks open the Venmo web page) instead of doing nothing.
