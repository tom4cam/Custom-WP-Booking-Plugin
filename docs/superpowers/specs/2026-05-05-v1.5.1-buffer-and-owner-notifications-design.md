# v1.5.1 — Buffer-time fix + configurable owner email — Design

**Date:** 2026-05-05
**Status:** Approved
**Scope:** Two small, independent fixes shipping together as v1.5.1.

## Goal

1. Remove the misleading "No slots will be bookable" admin error that
   triggers when buffer-time ≥ shortest enabled session length.
2. Make the new-booking owner email toggleable and its destination
   address configurable, matching the existing pattern of the owner
   SMS toggle/phone pair.

## Non-goals

- New cancellation/reschedule notifications for the owner (only
  "new booking" is in scope).
- Owner email template editor (uses the same HTML body as the client
  confirmation, with a `[New Booking]` subject prefix — current
  behavior).
- Owner WhatsApp channel.
- Backfilling tests for `class-notifications.php` (heavy mocking
  cost, low value — covered by manual smoke test).

## Change 1 — Drop the misleading buffer-time error

**File:** `includes/class-admin.php`

Remove lines 172–183 (the `add_settings_error` block triggered when
`buffer_time >= smallest_enabled`). Slot generation in
`Caswell_Google_Calendar::windows_to_slots()` already handles this
case correctly — slots simply space out at `length + buffer`
intervals. The error message claims slots can't be generated, which
is false for the practical 15+15 case.

The existing `min="0" max="60"` on the buffer-time input element
remains as the only validation.

Keep the line that sets `$clean['buffer_time']`. Only the
warning/error block is removed.

## Change 2 — Owner email: toggle + configurable address

### New settings

| Key | Type | Default | Notes |
| --- | --- | --- | --- |
| `owner_notify_email` | int (0/1) | `1` | Toggle. Default-on preserves current behavior on upgrade. |
| `owner_notify_email_address` | string (email) | `''` | Empty falls back to `get_bloginfo('admin_email')`. |

Naming mirrors the existing pair (`owner_notify_sms`,
`owner_notify_phone`).

### Sanitizer additions (`includes/class-admin.php`, near existing
`owner_notify_sms` keys)

```php
$clean['owner_notify_email']         = ! empty( $input['owner_notify_email'] ) ? 1 : 0;
$clean['owner_notify_email_address'] = sanitize_email( $input['owner_notify_email_address'] ?? '' );
```

### Admin UI (`admin/admin-page.php`, in the existing "Owner
Notifications" block in the SMS / Twilio tab, around lines 385–398)

Add **above** the existing "Send SMS to owner" line:

```
Send email to owner when a new booking is made    ← new
```

Add **after** the existing "Owner Phone Number" row:

```
Owner Email Address (optional, falls back to WordPress admin email)
```

The email-toggle and email-address rows reuse the same row markup
patterns already in use for the SMS pair.

### Refactor of `send_email` (`includes/class-notifications.php`,
lines 230–245)

Replace the current hardcoded admin-email send with a settings-aware
block:

```php
if ( 'confirmation' === $type && caswell_get_option( 'owner_notify_email', 1 ) ) {
    $owner_email = caswell_get_option( 'owner_notify_email_address', '' ) ?: get_bloginfo( 'admin_email' );
    $same_inbox  = $owner_email && strcasecmp( trim( $owner_email ), trim( $booking->email ) ) === 0;
    if ( ! $same_inbox ) {
        $owner_sent = wp_mail(
            $owner_email,
            "[New Booking] {$booking->name} — " . wp_date( 'M j, Y g:i A', caswell_local_ts( $booking->start_datetime ) ),
            $html,
            $headers
        );
        caswell_log( 'email', $owner_sent ? 'Sent owner alert email' : 'FAILED to send owner alert email', [
            'owner_email' => $owner_email,
        ] );
    }
}
```

Behavioral changes vs. current:
- Honors the new toggle.
- Sends to `owner_notify_email_address` if set, else WP admin email.
- Same de-dup against the client email (no double-send when Ryan
  books himself).
- Same restriction to `confirmation` type only — reminders are still
  client-only.

## Migration / compatibility

None. Existing installs without `owner_notify_email` saved get the
default `1` (on) and the empty address falls back to WP admin email
— exactly the current behavior.

## Verification

Manual smoke test on the live site after Dropbox sync:

- Buffer-time settings: enable 15-min, set buffer to 15, save. No
  red error. Settings save cleanly.
- Booking flow: book a 15-min slot. Confirm slots are generated
  every 30 minutes (length + buffer).
- Owner email: with the new toggle on and address blank, book a
  session — owner email arrives at WP admin email. With the address
  filled in, owner email arrives at that address instead.
- Owner email off: book a session — only the client email arrives.
- Owner SMS: existing behavior unchanged. With toggle on + phone
  set, SMS still arrives.
