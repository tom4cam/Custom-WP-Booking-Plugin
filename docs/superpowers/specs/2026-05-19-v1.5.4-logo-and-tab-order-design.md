# v1.5.4 — Site logo + Bookings tab first — Design

**Date:** 2026-05-19
**Status:** Approved
**Scope:** Add an admin-configurable site logo that renders on the home
page nav, booking widget, confirmation/reminder emails, and the admin
settings page. Also reorder the admin tabs so **Bookings** is the
leftmost tab and the default view.

## Goal

1. Let the owner upload a single logo image once (via the WordPress
   media library picker) and have it render — at appropriate sizes —
   on every customer-facing surface and the admin page. White-label
   friendly: no hardcoded paths or assumptions about Ryan's brand.
2. Surface the **Bookings** list (the operational daily-use tab) as
   the default landing tab on Settings → Caswell Booking, instead of
   the current Google Calendar tab.

## Non-goals

- Multiple logo variants (light/dark, mobile/desktop). One image, four
  rendering contexts, CSS scales it.
- A dedicated favicon setting. WP has site-icon support already.
- Animated logos or SVG-specific handling. PNG/JPG/WEBP/SVG via the
  media library all work — they're just `<img src>`.
- Repositioning, alignment, or wrap-options for the logo. Each context
  has one defined placement.

## New settings

Two new options stored on the existing `caswell_settings` blob
(no schema change):

| Key | Type | Sanitizer |
|---|---|---|
| `branding_logo_url` | URL string | `esc_url_raw` |
| `branding_logo_id` | int (attachment ID) | `absint` |

Both default to empty. The plugin treats `branding_logo_url` as the
source of truth at render time; `branding_logo_id` exists only so the
media-library picker can restore the "selected" thumbnail state when
the admin reopens the page.

## Admin UI — Business Info tab

A new row in the existing `Business Info` table:

```php
<tr>
    <th><label>Site Logo</label></th>
    <td>
        <div class="caswell-logo-picker">

                <?php endif; ?>
            </div>
            <p>
                <button type="button" class="button" id="caswell-logo-choose">Choose Logo</button>

            </p>
            <p class="description">Shown on the home page, booking widget, confirmation/reminder emails, and this admin page. Recommended: a transparent-background PNG or SVG sized around 200×60.</p>
        </div>
    </td>
</tr>
```

A small JS file (`admin/admin-logo-picker.js`) wires the buttons to
the WP media picker:

```js
jQuery(function ($) {
    var frame;
    $('#caswell-logo-choose').on('click', function (e) {
        e.preventDefault();
        if (frame) { frame.open(); return; }
        frame = wp.media({
            title:    'Choose Logo',
            button:   { text: 'Use as Logo' },
            library:  { type: 'image' },
            multiple: false,
        });
        frame.on('select', function () {
            var att = frame.state().get('selection').first().toJSON();
            $('#branding_logo_url').val(att.url);
            $('#branding_logo_id').val(att.id);
            $('.caswell-logo-preview').html(
                '<img src="' + att.url + '" alt="" style="max-height:80px;max-width:300px;" />'
            );
            $('#caswell-logo-remove').show();
        });
        frame.open();
    });
    $('#caswell-logo-remove').on('click', function (e) {
        e.preventDefault();
        $('#branding_logo_url').val('');
        $('#branding_logo_id').val('');
        $('.caswell-logo-preview').html('<span class="description">No logo selected.</span>');
        $(this).hide();
    });
});
```

`class-admin.php` `enqueue_admin_assets()` adds `wp_enqueue_media()`
and registers the script. Both happen only on the plugin's own admin
page (guarded by the existing screen check).

## Helper

`includes/branding.php` (new file, loaded from `caswell-booking.php`):

```php
<?php
defined( 'ABSPATH' ) || exit;

/**
 * Get the configured site logo URL, or '' if none.
 *
 * Returns the raw URL so callers can wrap it in <img> with their own
 * sizing classes / inline styles.
 */
function caswell_branding_logo_url() {
    $url = caswell_get_option( 'branding_logo_url', '' );
    return is_string( $url ) ? trim( $url ) : '';
}
```

Pure, easy to test with Brain Monkey.

## Tab reordering — Bookings first

`admin/admin-page.php` line 13-25 currently lists tabs in this order:

```
Google Calendar (active) → Sessions → Availability → Square → Venmo
  → Email → SMS / Twilio → Scheduling → Business Info → Bookings → Tools
```

After:

```
Bookings (active) → Google Calendar → Sessions → Availability → Square
  → Venmo → Email → SMS / Twilio → Scheduling → Business Info → Tools
```

Concretely:

- Move `<a href="#tab-bookings" class="caswell-tab active">Bookings</a>` to be the first item in the `<nav class="caswell-tabs">` block.
- Remove `active` from `<a href="#tab-gcal" ...>`.
- In the content section, move the `active` class from `<div id="tab-gcal" class="caswell-tab-content active">` to `<div id="tab-bookings" class="caswell-tab-content active">`.
- The actual content-div order can stay the same — JS toggles by ID, not source order. Reordering the nav links is enough for visual order.

## Migration / compatibility

- No DB schema change.
- No new dependencies.
- Installs that don't set a logo see **zero visible change** on the
  customer-facing surfaces. The tab reorder is the only difference,
  and it's reversible by simply re-ordering the nav anchors.
- The default landing tab changes from Google Calendar to Bookings.
  Reasonable for the daily-use case.
